# UI quality baseline

Start from existing product evidence: design system, components, layout conventions, accessibility requirements, and user-visible behavior.

Check:

- spacing and alignment use existing scale or explicit visual rationale
- typography hierarchy supports scan path
- color contrast meets the target platform standard
- interaction states cover hover, focus, active, loading, empty, and error states when relevant
- motion uses transform and opacity for smooth UI transitions
- data and copy match current product behavior
- third-party UI or motion libraries are present in project dependencies before use

Final output names visual behavior, accessibility checks, changed files, and validation evidence.
