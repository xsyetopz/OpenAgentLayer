# Localization-Ready Prose

Use this for prose that will be translated or read by people from many language backgrounds.

## Rules

- Keep the target language unless the user asks for translation.
- Avoid idioms and jokes.
- Avoid metaphors based on sports, schools, food, or local culture.
- Avoid phrasal verbs when a direct verb works.
- Keep noun phrases short.
- Keep lists parallel.
- Put conditions before actions.
- Use visible subjects. Avoid hidden subjects like "it" when unclear.
- Do not split a technical term across sentences.

## Non-English Text

When editing non-English prose:

- Preserve the language.
- Preserve formal/informal address unless the user asks to change it.
- Preserve product names and source language terms.
- Prefer common words over learned or literary words.
- Avoid translating technical terms if the language community uses the English term.
- If a sentence is ambiguous, state the ambiguity instead of guessing.

## UI Text

- Prefer verb + object: "Save file", "Run tests", "Copy link".
- Use the same label for the same action.
- Keep error text practical: what happened, why it matters, what to do next.
- Avoid blame. Say what the app could not do.

## Status Copy

Use this order:

1. What changed.
2. What the reader can do now.
3. What still does not work, if relevant.
4. What to do next.
