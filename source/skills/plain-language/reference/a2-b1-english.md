# A2-B1 English Copy

Use this when the target prose is English and the reader may use English as a second or third language.

## Sentence Rules

- Aim for 8-16 words per sentence.
- Use one clause when possible.
- Put the action near the start.
- Use direct verbs: use, set, run, open, choose, copy, save, send, build, test.
- Prefer names over pronouns when a pronoun can point to two things.
- Split cause and effect into two sentences when the sentence gets long.

## Word Choice

Use concrete words:

- value
- name
- file
- edge
- form
- rule
- command
- package
- page
- reader
- app
- code
- error

Avoid abstract words unless they are the technical term.

| Rough wording          | Prefer                                |
| ---------------------- | ------------------------------------- |
| surface                | area, API, rule, page                 |
| boundary               | edge                                  |
| domain                 | topic, app, work area                 |
| explicit               | clear, written down, visible          |
| implicit               | hidden, automatic                     |
| canonical              | main                                  |
| metadata               | extra data, route data, page data     |
| diagnostics            | error messages                        |
| semantic               | meaning                               |
| implementation details | how it works                          |
| temporary              | local, short-lived, close to the code |
| might                  | can, may, or a direct verb            |
| usually                | often, in most code, or a direct rule |
| utilize                | use                                   |
| leverage               | use                                   |
| facilitate             | help, make possible                   |
| ensure                 | make sure, check, keep                |

## Structure

- Start with what the reader needs to do.
- Then state the result.
- Then add the reason if the reason helps the reader decide.
- Keep repeated sections in the same order.

## Checks

After editing, ask:

- Can a reader translate each sentence in their head?
- Does each sentence carry one idea?
- Did I use the same word for the same idea?
- Did I keep every technical name exact?
