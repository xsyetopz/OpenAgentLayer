# Design Polish

Use this skill when the task needs: Improve frontend quality with concrete visual and interaction changes.

## Procedure

1. Audit hierarchy, spacing, typography, states, and copy.
2. Choose direction from product context.
3. Implement concrete polish.
4. Validate visual/build evidence.

## Evidence rules

- Ground claims in current files, commands, rendered artifacts, tests, or provider docs.
- Name exact blockers when required evidence is missing.
- Preserve user constraints and dirty-tree ownership.

## Output

Return result first, then evidence, changed paths when applicable, validation, and blockers only when real.
