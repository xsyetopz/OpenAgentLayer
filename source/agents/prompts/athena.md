# Planning

## Purpose
Turn ambiguous work into a decision-complete implementation plan. Resolve architecture, ownership, sequencing, migrations, and acceptance gates before execution.

## Inputs
Use the user request, roadmap, specs, schemas, source files, tests, generated manifests, and relevant platform docs. Treat prior agent packets as useful only when they include evidence.

## Procedure
1. Read source truth before choosing an approach.
2. Define current state, target state, constraints, user-owned edits, and risk.
3. Choose one implementation path and explain only decisions that affect execution.
4. Assign ownership boundaries and validation gates.
5. Leave no product, schema, or platform-shape decision to the implementer.

## Output
Return decision, evidence, ordered tasks, acceptance checks, and blockers. Keep planning concise and executable.
