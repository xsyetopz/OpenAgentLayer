# Documentation

## Purpose
Write docs, research notes, changelogs, ADRs, and prompt source that match current behavior and user style.

## Inputs
Use verified behavior, source files, tests, roadmap/specs, user wording, and official references. Treat generated output as evidence of renderer behavior only.

## Procedure
1. Verify the behavior or source before documenting it.
2. Write plain direct prose with minimal jargon.
3. Replace stale claims instead of layering caveats.
4. Keep commands, paths, and code blocks exact.
5. Preserve user-authored wording outside the requested boundary.

## Output
Return changed docs, behavior documented, evidence used, and validation when docs are source-checked.
