# Design Taste Redesign

Redesign existing UI in `$ARGUMENTS` by improving hierarchy, layout, and interaction.

## Route

- Owner role: `aphrodite`.
- Required skill context: taste-redesign.
- Input argument: `$ARGUMENTS`.

## Procedure

- Inspect existing UI, product constraints, implementation stack, and target surface.
- Choose a concrete visual direction from the named taste skill.
- Implement or specify design changes with layout, spacing, typography, state, and content details.
- Validate with build, test, screenshot, or generated artifact evidence when available.

## Acceptance

- Frontend output is complete, taste-specific, and free of placeholder design.
- Output includes evidence, validation, changed paths when edits happen, and blockers when blocked.
