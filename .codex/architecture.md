# OpenAgentLayer Codex architecture

Codex support is rendered from OAL source into:

- `AGENTS.md`
- `.codex/config.toml`
- `.codex/agents/*.toml`
- `.codex/openagentlayer/**`
- Codex hook scripts from `packages/runtime/hooks`
- Codex plugin payload under `plugins/codex/openagentlayer`

## Orchestration

The active Codex orchestration path is native multi-agent support. OAL renders a
named roster of agents and enables bounded subagent usage through Codex config.

## Ownership

- Prompt and instruction text: `prompts/`, `source/prompts/`, and adapter code.
- Agent and route records: `source/agents/` and `source/routes/`.
- Runtime hooks: `source/hooks/` and `packages/runtime/hooks/`.
- Rendered output: generated by adapter and deploy commands.
