# OpenAgentLayer Instructions

This project is managed by OAL for opencode. Treat authored source as the source of truth and generated artifacts as disposable outputs. Use provider-native capabilities only when OAL renders and validates them. Do not modify generated files by hand; update source and regenerate.

Routes:
- accept: Run full acceptance in clean fixture roots without touching user home. Acceptance must fail on stubs, bad models, drift, non-executable hooks, and metadata-only tools. The route must declare owner, permissions, allowed tools, validation expectations, output contract, hook expectations, and provider-specific differences. It is invalid if reduced to a two-line descriptor.
- audit: Audit product behavior against roadmap, provider surfaces, manifest ownership, source boundaries, and safety guardrails. The route must declare owner, permissions, allowed tools, validation expectations, output contract, hook expectations, and provider-specific differences. It is invalid if reduced to a two-line descriptor.
- debug: Reproduce failures, minimize cases, inspect exact evidence, and identify root cause before changing code. The route must declare owner, permissions, allowed tools, validation expectations, output contract, hook expectations, and provider-specific differences. It is invalid if reduced to a two-line descriptor.
- design: Improve UI or product surface quality with accessibility, visual, and interaction evidence. The route must declare owner, permissions, allowed tools, validation expectations, output contract, hook expectations, and provider-specific differences. It is invalid if reduced to a two-line descriptor.
- document: Update documentation only when it describes implemented behavior. Keep examples tied to real CLI commands and generated artifacts. The route must declare owner, permissions, allowed tools, validation expectations, output contract, hook expectations, and provider-specific differences. It is invalid if reduced to a two-line descriptor.

The route must declare owner, permissions, allowed tools, validation expectations, output contract, hook expectations, and provider-specific differences. It is invalid if reduced to a two-line descriptor.
