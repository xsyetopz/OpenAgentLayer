# OpenAgentLayer v4 plans

Purpose: planning authority for the v4 reboot work. These files tell future AI agents what to build, in what order, and how to mark progress.

Authority: `specs/` defines final behavior. `plans/` defines execution order. `docs/` provides evidence and studies.

Status markers:

- `[ ] Queued` means not started.
- `[~] Active` means under work now.
- `[x] Sealed` means complete and evidence-linked.

Rules for agents:

- Define v4 behavior directly.
- Do not preserve prior names or surfaces unless a v4 spec explicitly defines them.
- Treat v3 as research input only.
- Update checkboxes when work changes state.
- Add evidence next to any sealed item.
- Keep plan files operational, not motivational.
