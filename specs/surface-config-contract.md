# Surface config contract

Purpose: define how OpenAgentLayer adapters consume upstream surface config schemas.

Authority: normative companion to `openagentlayer-v4.md`.

## Supported surfaces

This contract currently covers:

- Codex;
- Claude Code;
- OpenCode.

Other surfaces are outside this contract until their studies are added.

## Adapter config allowlists

Each adapter must maintain a schema-derived allowlist:

- allowed top-level keys;
- allowed nested keys;
- unsupported current keys with reason;
- keys that must not be emitted;
- replacement mapping for any upstream key that points at a newer key.

Render fails when OAL attempts to emit a key outside the adapter allowlist.

## Default profiles

Each adapter must define a `default_profile` record for each supported surface.

Required fields:

- `surface`: `codex`, `claude`, or `opencode`;
- `profile_id`: generated profile name or equivalent surface selector;
- `placement`: config placement class;
- `emitted_keys`: exact keys the adapter writes;
- `source_url`: upstream schema/docs URL for each emitted key group;
- `validation`: parser/schema checks required after render.

Default profiles are data. Renderers must not hide defaults in string templates.

## Config placement classes

Every emitted config key must have one placement class:

- Generated project profile: team-safe repo behavior.
- Generated global profile: user-approved global OAL behavior.
- User-local recommendation: personal preference that OAL documents but does not overwrite during project install.
- Policy-owned route override: route contract, hook policy, permission policy, or model policy value.
- Runtime-owned generated: path or command pointing at files created by OAL installer.
- Enterprise-managed note: organization policy documentation only.

Project install may write only generated project profile, policy-owned route override, and runtime-owned generated values. Existing user-owned config keys must not be overwritten; OAL-owned structured keys are tracked by manifest path/value metadata.

Global install may write generated global profile values only when explicitly selected.

User-local recommendations must be rendered as docs or dry-run output, not silently written.

## Replacement mapping

When an upstream schema or official docs identify a newer key, OAL must emit only the newer key.

Required replacement records:

- Codex approval fallback policy -> `approval_policy = "on-request"` for interactive routes or `approval_policy = "never"` for non-interactive routes.
- Codex old approval reviewer name -> `approvals_reviewer = "auto_review"`.
- Claude old commit attribution toggle -> `attribution`.
- Claude old voice boolean -> `voice.enabled`.
- OpenCode old sharing boolean -> `share`.

## Config ownership

Adapters must classify each emitted config field:

- OAL-owned: generated by source graph.
- User-owned: never overwritten by OAL.
- Merge-owned: OAL may add/remove only inside managed block or manifest-owned object.
- Policy-owned: generated from route contract or hook policy source.

Each owned field must also record:

- upstream key name;
- upstream source URL;
- placement class;
- validation rule;
- renderer responsible for writing it.

Installer ownership modes:

- full-file: generated artifacts owned entirely by OAL;
- marked-text-block: Markdown/text files where OAL owns only the marked block;
- structured-object: JSON/TOML files where OAL owns only manifest-listed key paths.

## Validation

Each adapter must validate:

- generated config parses;
- generated keys are in the allowlist;
- no key from the do-not-emit table is present;
- replacements are used where required;
- generated config matches install scope.
- generated defaults match the surface `default_profile`;
- user-local recommendations are not written by project install;
- runtime-owned paths exist after install.
