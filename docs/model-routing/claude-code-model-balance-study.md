## OAL Claude subscription support

OAL generated Claude presets target:

- `claude-max-5`
- `claude-max-20`

This is an OAL support boundary. Official Claude Code docs also describe other account types, but OAL presets focus on Max 5x/20x.

## Alias model policy

Claude Code supports aliases and full model names.

| Alias | Current meaning for Anthropic API | OAL use |
| --- | --- | --- |
| `default` | account-dependent recommended model | Do not use in generated OAL presets. |
| `best` | strongest available model, currently equivalent to `opus` | Avoid in generated presets; too implicit. |
| `haiku` | fast efficient Haiku for simple tasks/background work | Use for small helpers where exact pin is not needed. |
| `sonnet` | latest Sonnet; current Anthropic API resolution is Sonnet 4.6 | Default implementation/execution alias when moving-current behavior is desired. |
| `opus` | latest Opus; current Anthropic API resolution is Opus 4.7 | Strategy/review/orchestration alias when moving-current behavior is desired. |
| `opusplan` | `opus` in plan mode, `sonnet` in execution mode | Good default for primary interactive routes on Max 5x/20x. |
| `sonnet[1m]` | Sonnet with 1M context where available | Explicit long-context route only. |
| `opus[1m]` | Opus with 1M context where available | Explicit long-context strategy/review route only. |

## Exact model pinning

When OAL wants exact model behavior, it must emit a full model ID.

Rules:

- To use Opus 4.7 exactly, emit `claude-opus-4-7`.
- To use Sonnet 4.6 exactly, emit `claude-sonnet-4-6`.
- To use Opus 4.6 exactly, emit `claude-opus-4-6`; do not emit `opus`.
- To use 1M context with an exact model, append `[1m]` where the model supports it, for example `claude-opus-4-6[1m]`.
- Full model ID is mandatory for any route that must not drift when Claude aliases update.

## Environment variable pinning

Claude Code supports alias remapping through full model names:

- `ANTHROPIC_DEFAULT_OPUS_MODEL`
- `ANTHROPIC_DEFAULT_SONNET_MODEL`
- `ANTHROPIC_DEFAULT_HAIKU_MODEL`
- `CLAUDE_CODE_SUBAGENT_MODEL`

OAL use:

- Use settings `model` for route/profile-level defaults.
- Use `ANTHROPIC_DEFAULT_*` when the generated environment intentionally pins aliases for a workspace.
- Use `CLAUDE_CODE_SUBAGENT_MODEL` when all subagents should use a cheaper or safer default.
- Do not use old small-fast environment names.

## Effort levels

Claude Code adaptive effort levels:

- Opus 4.7: `low`, `medium`, `high`, `xhigh`, `max`.
- Opus 4.6 and Sonnet 4.6: `low`, `medium`, `high`, `max`.

OAL defaults:

- No `max` default.
- No `xhigh` default.
- `low`: small Haiku helper routes.
- `medium`: docs, research, validation, bounded implementation.
- `high`: planning, review, implementation requiring careful reasoning.
- `xhigh`: Opus 4.7 explicit deep route only.
- `max`: manual-only; not generated by default.

## Extended context

Claude Code supports 1M context variants where plan/account/model allow:

- `opus[1m]`
- `sonnet[1m]`
- full model IDs with `[1m]`, such as `claude-opus-4-6[1m]`.

OAL rules:

- 1M context is a route modifier, not default.
- Use 1M for huge repo studies, generated-output audits, cross-surface schema ingestion, or long-session compaction recovery.
- Do not use 1M for routine implementation, reviews, or tests.
- Document the route reason whenever a 1M variant is selected.

## Plan: `claude-max-5`

Purpose: usage-aware Max 5x preset.

Primary model choices:

- interactive default: `opusplan`;
- implementation subagents: `sonnet` or `claude-sonnet-4-6`;
- small helpers: `haiku`;
- exact Opus 4.6 route: `claude-opus-4-6`;
- 1M route: explicit only.

Effort:

- default interactive: `medium` or `high`;
- planner: `high`;
- implementer: `medium`;
- reviewer: `high`;
- helpers: `low` or `medium`;
- no default `xhigh` or `max`.

## Plan: `claude-max-20`

Purpose: higher-capacity Max preset.

Primary model choices:

- interactive strategy/orchestration: `opus` or `opusplan`;
- implementation: `sonnet` or `claude-sonnet-4-6`;
- small helpers: `haiku`;
- explicit exact Opus route: `claude-opus-4-6` or `claude-opus-4-7`;
- long-context strategy/review: `opus[1m]` or exact full ID with `[1m]`.

Effort:

- default interactive: `high`;
- planner/reviewer: `high`;
- rare deep route: `xhigh` only on Opus 4.7;
- no default `max`;
- helper roles stay `low`/`medium`.

## Settings optimization

OAL Claude settings should:

- set `model` only when the adapter owns the route default;
- use `availableModels` to restrict generated choices where needed;
- use `env` for exact alias pins only when OAL owns workspace model routing;
- use subagent/skill frontmatter effort for role-local tuning where supported;
- use command hooks for policy runtime;
- keep project settings team-safe;
- keep personal credentials and machine-specific provider config out of shared project settings.

## OAL implementation notes

- Prefer aliases for moving-current Max presets.
- Prefer full model names for exact study/review routes.
- Make `haiku` first-class for cheap helper roles.
- Use `CLAUDE_CODE_SUBAGENT_MODEL=haiku` only for subagent classes that are guaranteed not to need coding/review quality.
- Give each Greek role its own model policy instead of one global subagent model when the surface supports it.
