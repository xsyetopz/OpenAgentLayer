# v3 Success/Failure Matrix

| Area                 | v3 success                                                                                  | v3 shortfall                                                                                                | OAL reboot requirement                                                                                                  |
| -------------------- | ------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------- |
| Source model         | Authored source under `source/` generated platform targets.                                 | Source/generated/deploy boundaries were still blurry.                                                       | Separate authored source, generated output, deployable payload, and installed manifest state.                           |
| Agent prompts        | Full operating manuals with constraints, protocols, and output formats.                     | Prompt rendering lived inside a broad generator; model/tool contracts spread across metadata and rendering. | Keep operational depth; make prompt body, model route, tool contract, provider support, and generated output traceable. |
| Commands/routes      | Route kind fields existed; Codex commands carried contract flags.                           | Per-provider command catalogs encouraged duplication and shallow provider-specific cards.                   | Define OAL routes once with provider overlays; render native provider command surfaces.                                 |
| Hooks                | Executable `.mjs`; stop-scan enforced real behavior.                                        | Hook generation and runtime packaging were entangled with broad generator/install logic.                    | Preserve executable hooks; separate hook policies, provider bindings, packaged runtime, and fixtures.                   |
| Provider differences | Some capabilities explicitly unsupported, such as Codex lacking Claude-style SubagentStart. | Not systematic enough across all provider surfaces.                                                         | Capability matrix for every OAL concept per provider. Unsupported is explicit.                                          |
| Installer            | Real multi-provider installation with optional integrations.                                | Too much imperative behavior in one CLI and unclear manifest ownership.                                     | Render → plan → dry-run/diff → apply → manifest → uninstall.                                                            |
| Model routing        | Plan-aware routing and swarm limits existed.                                                | Uses now-disallowed model set and xhigh-heavy choices.                                                      | Enforce current allowed model sets and route-aware cost control.                                                        |
| Integrations         | Caveman, RTK, Context7, DeepWiki, Playwright were product features, not just docs.          | Integration scope and provider boundaries were mixed into installer logic.                                  | Integrations are scoped OAL capabilities with provider-specific render/deploy behavior.                                 |
| Validation           | Tests and generated checks existed.                                                         | Product acceptance did not fully prove deploy/uninstall/hook/tool/provider behavior end-to-end.             | Acceptance must prove source load, render, deploy, uninstall, hooks, tools, model policy, and v3 isolation.             |
| Branding             | openagentsbtw coherent for v3.                                                              | Reboot must not mention “oabtw v4.”                                                                         | Product language is OpenAgentLayer / OAL only. v3 reference stays in `v3_legacy`.                                       |
